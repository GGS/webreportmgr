19.03.2015
Очень много исправлено. Перенес процесс отображения таблицы на
клиента - теперь таблица собирается в браузере, сервер только шлёт
бинарные данные.
Расширил таблицу report.
Сделал dets копию report. Во время запуска проверяется её наличие на
диске и при обнаружении все данные копируются в ets. Если произошёл
крах программы, то все работающие задания имеею возможность
перезапуститься, для этого введен новый статус restoring. Пока
перезапуск сделан в ручном режиме.
Время выполнения теперь вычисляется из данных времени которые хранятся
в report. Для этого после успешной генерации отчёта поле имени файла
заменяется на метку времени. 
5.03.2015
 Возможность убивать процесс. Для это при запуске задания
 пид внешней программы записывается в отдельную ets таблицу по
    идентификатору задания. При получении запроса на остановку
    процесса вызывается системный вызов kill. 
    Также сделал
    отслеживание функционирования очереди. Если основной процесс
    падает, то программа его перезапускает. Опрос через poll.
4.03.2015
Вывод данных по типу и размеру сгненерированных файлов.
03.03.2015
Сделал проверку на дублирование данных. Клиент генерирует хэш а на
стороне сервера происходить сравнене, если хэш есть в в etc, то данные
не засылаются и клиент посылает запрос на получение отчёта. Хэш с
этого обновления будет действовать как идентификатор задания.
Сдела функцию аптайма, время работы приложения без перезагрузки. Всё
находиться в модуле poll.
